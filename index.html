Basic JavaScript Terms

Hoisting: JavaScripts default behavior of moving all declarations to top of current scope(script/function); declare variables at the beginning of script.
Identifiers: Used to name Variables (Keywords, Functions, Labels).
Parameter: Objects data properties or values.
Literals: Actual data rather then from a calculation or variable.
Expression: Values, Variables and Operators; witch computes to a value.
Statements: A JavaScript program is a list of statements to be executed by a computer, often called code. Statements are separated with a semicolon; Often start with keyword to identify action to be performed. Programmers like to avoid code lines(statements) more than 80 characters, best place to break is after an operator
Parsing: JavaScript interpreter works from left to right; like reading.
Comments: // - Used to leave comments inside of the Script Code not displayed (for programming use). Also used to prevent execution, any text between // and the end of the line will not be executed; Multi-line Comments: start with /* and end with */.
Data types: JavaScript uses [String, Number, Boolean, Undefined, Function, Object, Arrays] Data type Conversions: Can Convert data types from one to another.
External Link: Using the Script element through src attribute - <script src="aScriptFile.js"> Can source external files for use in Script. Used for JavaScripts Maintainability, Browser Speed and Compatibility, usually ends with .js.
Debugging: The process of searching code to locate a source of trouble.
