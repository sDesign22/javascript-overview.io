1.5 Functions

Are a block of code designed to perform a particular task and executed when Invoked (called); Functions are very useful for repetitive tasks. Instead of writing code every time, functions can be called multiple times to get the same task done with different results. Functions are a very useful and easy to use, they even create variables without declaring prior(with var keyword).

Functions Syntax

Defined with the function keyword, followed by a name(same rules as variables), followed by parentheses ():

function myFunction(parameter1, parameter2) { return parameter1 + parameter2; } // This is a basic functions showing the results of the parameters multiplied together.

Parentheses(parameter1, parameter2) // Data that a Function requires to be passed are its Parameters; May include parameter names separated by commas. Parentheses are required (sets apart functions).
Body of the Function: Code to be executed, by the function, is placed inside curly brackets: { Code to be executed; } Everything between the brackets is part of the function.
return: Statement stops execution of a function and returns a value from that function. return value; If function was invoked from a Statement.

Function Invocation Code in function executes when "something" invokes (calls) the function: When an Event occurs or invoked (called); can also Automatically (self invoked).

functionName(argumentsValues) // Function Call Syntax.

Arguments are the values received by the function when it is invoked. Inside the function, the arguments (the parameters) will behave as local variables.

Function Methods


Functions Can be Values:

var newValue = aFunctionAsValue(); // This assigns a function to a variable.

Function Properties

Function Object "toCelsius()" Refers to the function result. "toCelsius" Refers to the function object; Accessing a function without () returns definition.

Scope and Lifetime:
Scope: JavaScript has function scope, Each function creates a new scope. Scope determines the accessibility (visibility) of these variables. Variables defined inside a function are not accessible (visible) from outside the function. Determined by where a variable is declared and whether the var keyword is used or not (Identifier Lookup).
Global or Local
1. Global Variables: A Variable that is declared Outside any functions is Global. The global variable can be referenced anywhere in the code.
2. Local Variables: A Variable that is declared Inside a function is Local. Can only be referenced within the function it is declared. This variable does not exist outside the function.
Lifetime: Starts when it is Declared. Local variables are deleted when the function is completed. In a web browser, global variables are deleted when you close the browser window (or tab).
